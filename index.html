为什么要去潜水？

发表于 2017-03-16

从结完婚一个月后就去菲律宾考了 OW，而不是和大家一样结完婚就去蜜月开始。学潜水的路就充满了不理解和坎坷。甚至到后面几次出去刷瓶我已经不让家人知道了，只是说我要出去玩然后就和陶先生串通好帮我打掩护，保险单号留给他我就跑了。
其实想想我不是很负责的一个人，不只是不负责可能还比较自私。城市里繁琐的一切都被我逃避了大半，当初要买房到处看楼盘最后确定买的是陶先生，那时候我还在 PG 岛。白天抗装备3潜晚上和陶先生视频，’房买好了。’买哪了多大’,’松江，90平不到，挺好的。’ 然后我就跟着他抗下了这栋房。后来拿房的时候也是，都是陶先生自己搞定的，其实想想我真的不是一个好妻子好家人，我还觉得自己没看够外面的很多很多。不甘于现状又无力摆脱。就像别人说的：人的一切痛苦都是对自己无能的愤怒。

那么回到主题：为什么要去潜水？

如果用那句装逼金句“因为山在那里”套成“因为大海在那里”，显然属于没话找话。
喜欢去潜水就很多人喜欢徒步、登山、滑雪一样，大概就是想去寻找除了自己熟悉的那一部分世界之外的世界，有一种“哦，原来世界还可以这样”的感悟。
在水下面对的一切，你所感知到的一切，以一种不可复制的形式存入你个人的大脑，变成别人想象不到的独家记忆。那一刻，那个熟悉的烦心的世界离我好远，自己终于可以放心享受无人打扰的自由。即使回到上海上重新加入挤地铁的人流里，心里也残留着小小的优越感：昨天，我可是“the king of the world”……

去那里，不是为了征服，而是为了更好的敬畏。

我希望以后我更多的去潜水不是为了逃避而是一种习惯。因为我知道现在或者近几年我还是在逃避。啊这样真的太痛苦了。

关于如何同时使用多个 Github 账户的方法

发表于 2017-03-09

为什么会有这个需求是因为工作上对很多团队的私有仓库要有访问权限，所以把电脑上的 ssh-key 和有权限的工作账号绑定，某天突然发现 pull 不了自己账号上的仓库了。

于是想到如何可以同时使用多个 Github 账户同时对应不同的 ssh-key 做到对多仓库的访问。

Step 1 - 为第二个账户新生成一个 ssh-key

ssh-keygen -t rsa -C "your-email-address"
这里要注意的是不要把第一个账户的 ssh-key 覆盖了，在提示保存时不要使用默认的 id_rsa 而是 id_rsa2 就好了。

Step 2 - 关联新的 ssh-key 和第二个账户

登录相应的账户 => Personal settings => SSH and GPG key
将生成的 id_rsa2 的新增进去。

Step 3 - 生成 Config 文件
